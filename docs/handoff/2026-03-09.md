## 작업 내용

### v2 프로젝트 구조 전환 (Step 1)

- v2 CLI 구조: `inbox` / `pack` / `run` / `note` / `legacy` 서브커맨드
- v2 config: SchoolSync, Daglo, NotebookLM, Obsidian 설정 분리
- requirements.txt 경량화 (torch/whisper/chromadb 제거 → pyyaml, loguru만)
- pyproject.toml 버전 0.3.0

### Phase 2 구현 (Step 2)
- `srt_parser.py` — 다글로 SRT 파싱, 세그먼트 병합, NotebookLM 포맷 변환
- `context_builder.py` → 삭제됨 (school_sync의 `context_export.py`가 직접 생성하는 구조로 전환)
- `packer.py` — 오케스트레이션 + `_load_context`로 school_sync context 파일 직접 읽기 + 폴더 자동 열기
- `guide_generator.py` — NotebookLM 가이드 프롬프트 자동 생성

### 추가 기능
- `inbox.py` — `input/daglo/inbox/`에 파일 넣으면 과목명/날짜 자동 감지 → 과목별 폴더로 분류. 감지 실패 시 CLI 입력
- school_sync `main.py`에 실행 로그(`output/.last_run.json`) 추가
- `packer.py`의 `_check_school_sync` — pack 시 전사본 날짜와 last_run 비교, 오래됐으면 자동 크롤링 실행
- `run` 서브커맨드 — inbox → pack 전체 워크플로우 원커맨드

### 감사에서 수정한 버그
- config.py: dead import(`os`), config mutation(`nlm.pop`), `sys.argv` 미복원
- inbox.py: 과목명 부분문자열 오매칭 → 길이 역순 정렬로 수정
- inbox.py: 날짜 패턴 월/일 범위 검증 없음 → 정규식 강화
- packer.py: `pack_all`이 `inbox/` 디렉토리도 순회 → skip_dirs로 제외
- guide_generator.py: 전사본↔PDF 매칭을 키워드 기반에서 **원문 대조 우선 + 키워드 보조** 방식으로 변경

## 핵심 결정 & 맥락

### 아키텍처 결정
- **context_builder.py 제거**: school_sync가 `context_export.py`로 `output/context/{과목}.md`를 직접 생성하는 구조로 전환됨. lesson-assist는 이 파일을 읽기만 함. 역할 분리가 더 깔끔.
- **레거시 보존**: v1 코드를 `legacy/`에 보존. `python -m lesson_assist legacy process --audio ...`로 v1 fallback 가능. 다글로/NotebookLM 접속 불가 시 대비.
- **가이드 프롬프트**: NotebookLM 소스로 함께 업로드되어 동작을 안내하는 문서. PDF 원문 대조 우선 매칭 방식 채택.
