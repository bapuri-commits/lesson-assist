# lesson-assist Handoff — 2026-03-06

## 현재 상태
Phase 2까지 구현 완료된 상태에서 **대규모 리팩토링** 수행. 디렉토리 구조를 과목/날짜 계층으로 재설계하고, 요약 버전 관리, 수업자료 명시적 지정, 대화형 교정 UX 개선, RunPod 전사 백엔드, school_sync 경로 연동을 모두 반영. 53개 테스트 전부 통과. 실제 녹음으로 E2E 테스트 대기 상태.

## 작업 내용
- **SessionDir 헬퍼 클래스** (`session.py` 신규): 모든 경로 로직을 중앙 관리. `pipeline.py` 및 모든 모듈(transcribe, review, segment, subtitle, summarize, actions)이 SessionDir에 위임.
- **디렉토리 구조 변경**: flat `data/{type}/{file_id}_*` → 계층 `data/{과목}/{날짜}/transcript_segments.json` 등
- **버전 관리**: 요약 `summary_v1.json`, `summary_v2.json`... 재실행 시 덮어쓰기 대신 새 버전 생성. 교정 반영 전사도 `transcript_segments_r1.json` 리비전 분리. 메타데이터(`version`, `based_on`) 추적.
- **수업자료 명시적 지정**: `--materials` CLI 옵션 → `materials.yaml` 저장 → RAG에 해당 자료만 로드. 지정 없으면 eclass fallback.
- **대화형 교정 UX**: 앞뒤 3개 세그먼트 문맥 표시, `[n/total]` 진행률, `a` 키로 나머지 전부 스킵.
- **RunPod 전사 백엔드**: `config.yaml`에 `backend: "local"|"runpod"` + `runpod:` 설정 추가. `transcribe.py`에 `_transcribe_runpod()` 구현 (base64 업로드 → 폴링 → 결과 수신). RunPod 엔드포인트 Docker 이미지는 아직 미세팅.
- **school_sync 경로**: `eclass.py`가 `raw/eclass/` 하위 탐색 우선 + 레거시 fallback 유지. `config.yaml`의 `data_dir`을 `school_sync/output`으로 변경.
- **exam_sheet.py 버그 수정**: 새 디렉토리 구조에서 요약 파일을 못 찾던 glob 패턴 수정 + 날짜별 최신 버전 선택 로직 추가.
- **마이그레이션 스크립트** (`migrate_data.py`): 기존 flat 데이터를 새 구조로 이동. `--dry-run` 지원.
- **테스트 추가**: SessionDir 11개 + config RunPod 2개 + eclass school_sync 1개 = 총 53개 통과.

## 핵심 결정 & 맥락
- **SessionDir 패턴 채택**: 경로를 각 모듈이 직접 조립하는 대신 SessionDir에 집중. 레거시 `save(out_dir, file_id)` 메서드는 유지하되 내부에서 `save_to()`를 호출하는 래퍼로 변환.
- **버전 네이밍**: `_v1`, `_v2`(요약), `_r1`, `_r2`(전사 리비전). glob 스캔으로 자동 번호 증가.
- **수업자료 우선순위**: CLI `--materials` > 세션 `materials.yaml` > eclass 자동 탐색. 사용자가 명시적으로 제어 가능.
- **RunPod**: 로컬 전사를 레거시로 유지하면서 `backend` 옵션으로 전환. 실제 RunPod 엔드포인트(Docker + handler.py)는 후속 작업.
- **RAG db_path**: `_get_rag_store()`에서 `cfg.rag.db_path`를 `session.rag_dir`(`data/_rag/`)로 직접 변경하는 side effect 있음. 파이프라인 1회 실행 내에서는 문제 없음.
