## 프로젝트 개요

수업 녹음 파일을 넣으면 **전사 → 요약 → Obsidian 노트**가 자동 생성되는 로컬 CLI 파이프라인.

### 문제

- 90분 강의를 수동 정리하면 30~60분 추가 소요
- 교수님이 판서/슬라이드를 가리키며 설명할 때 음성만으론 맥락이 끊김
- 과제·시험·일정 언급이 강의 중간에 흩어져서 놓치기 쉬움

### 해결

| 단계 | 동작 |
|------|------|
| 입력 | 녹음 파일(m4a/mp3/wav)을 CLI에 전달 |
| 전사 | 로컬 GPU로 faster-whisper 한국어 전사 (타임스탬프 포함) |
| 품질 검토 | 저신뢰 구간을 후보로 추출 → 사용자 교정 기회 제공 |
| 분할 | 긴 전사본을 ~25분 파트로 자동 분할 |
| 요약 | OpenAI API로 파트별 요약 → 통합 요약 생성 |
| 액션 추출 | 과제·시험·일정 언급을 자동 감지 |
| 노트 생성 | Obsidian 마크다운 파일 생성 + The Record 데일리 노트 연동 |

### 환경

- **PC**: Ryzen 9 3900X + RTX 3080 Ti (VRAM 12GB)
- **OS**: Windows 10/11
- **언어**: 100% 한국어, 교수 단독 발화 (화자 구분 불필요)
- **API**: OpenAI (GPT-4o 기본, 품질 최우선)
- **볼트**: Obsidian 로컬 (`G:\CS_Study\The Record`)

### 세 단계 비교

| 구분 | Phase 1: MVP | Phase 2: 확장 | Phase 3: 자동화 |
|------|-------------|---------------|----------------|
| 목적 | 개인 도구 (돌아가는 것) | 포트폴리오 (쓸만한 것) | 포트폴리오 (편한 것) |
| 전사 | faster-whisper + 저신뢰 후보 | + SRT/VTT 출력 | 동일 |

## Phase 1: MVP — 핵심 파이프라인

### 설계 철학

1. **품질 최우선**: 모델은 가장 좋은 것 사용 (whisper large-v3, GPT-4o). 비용보다 정확도
2. **사용자 교정 루프**: 전사가 의심스러운 구간은 반드시 후보로 제시. AI가 "맞다고 확신하는 것만" 최종 결과에 반영
3. **즉시 쓸 수 있는 것**: 설치 후 명령 하나로 노트 생성까지 완료

### 아키텍처

```
[녹음 파일] ─→ [전처리] ─→ [전사] ─→ [품질 검토] ─→ [분할] ─→ [요약] ─→ [노트 생성]
  m4a/mp3/wav    ffmpeg      faster-whisper   저신뢰 후보     ~25분 파트   OpenAI API    Obsidian MD
                 정규화       large-v3 (GPU)   추출/제시       자동 분할    파트별+통합   + 데일리 연동
                                                                          + 액션 추출
```

### 전사 (transcribe)

**엔진**: faster-whisper + large-v3

| 설정 | 값 | 이유 |
|------|----|------|
| model | large-v3 | 한국어 정확도 최우선 |
| language | ko | 자동 감지보다 명시가 안정적 |
| device | cuda | RTX 3080 Ti 활용 |
| compute_type | float16 | VRAM 절약 + 속도 균형 |
| beam_size | 5 | 기본값, 품질 유지 |
| vad_filter | true | 무음 구간 스킵으로 속도 향상 |
| word_timestamps | true | 세그먼트별 타임스탬프 확보 |

**출력물**:
- `{id}_raw.txt` — 전체 전사 텍스트 (plain)
- `{id}_segments.json` — 세그먼트별 텍스트 + 시작/끝 시간 + 신뢰도(avg_logprob, no_speech_prob)

### 품질 검토 (review)

faster-whisper는 세그먼트마다 `avg_logprob`(평균 로그 확률)과 `no_speech_prob`(무음 확률)을 제공한다.
이 값을 기준으로 **저신뢰 구간을 자동 후보화**하고, 사용자가 교정할 수 있도록 제시한다.

**후보 추출 기준** (임계값은 실사용 후 조정):

| 조건 | 기본 임계값 | 의미 |
|------|-------------|------|
| avg_logprob < threshold | -0.7 | 모델이 전사에 자신 없음 |
| no_speech_prob > threshold | 0.5 | 실제 발화가 아닐 가능성 |
| 세그먼트 길이 < 2자 | — | 의미 없는 잡음 전사 가능성 |
| 반복 패턴 | — | 동일 단어/구 3회 이상 반복 (hallucination 징후) |

**교정 워크플로우**:

```
1. 파이프라인 실행 → 전사 완료
2. 저신뢰 후보 목록 출력 (터미널 + review 파일)
   예:
   [1] 35:20-35:45 (logprob: -0.82) "???한 집합에서 ??? 연산을..."
   [2] 41:05-41:12 (logprob: -0.91) "이거를 쪼개면 그 뭐냐..."
   [3] 55:30-55:35 (no_speech: 0.72) "음..."
   
3. 사용자 선택:
   a) 교정 파일 편집 후 재실행 (--review 모드)
   b) 무시하고 진행 (--skip-review)
   c) 대화형 교정 (--interactive)
```

**교정 파일 포맷** (`{id}_review.jsonl`):
```json
{"idx": 1, "start": "35:20", "end": "35:45", "original": "???한 집합에서...", "corrected": "", "action": "pending"}
{"idx": 2, "start": "41:05", "end": "41:12", "original": "이거를 쪼개면...", "corrected": "", "action": "pending"}
```
- `action`: `pending` → 사용자가 `corrected`를 채우고 `accepted`로 변경, 또는 `skipped`

### 분할 (segment)

긴 전사본을 요약 가능한 크기로 나눈다.

**전략**:
- 기본 단위: ~25분 (오디오 시간 기준)
- 분할 지점: 세그먼트 경계 중 **가장 긴 무음 구간**에서 분할 (문장 중간 절단 방지)
- 최소 파트 길이: 5분 (너무 짧은 잔여 파트는 이전 파트에 병합)

**출력물**:
- `parts/part_01.txt`, `part_02.txt`, ...
- 각 파트에 시작/끝 타임스탬프 메타데이터 포함

### 요약 (summarize)

**모델**: GPT-4o (품질 최우선)

#### 1차 요약 — 파트별

각 파트 텍스트를 개별 요약한다.

**프롬프트 출력 구조**:
- 이 파트의 주제 (1줄)
- 목차 (섹션 5~10개)
- 핵심 개념/정의 (정확히)
- 정리/규칙/조건 (있으면 엄밀히)
- 예제/직관
- 교수 강조 포인트 ("시험에 나온다", "중요", "기억해라", "자주 나오는" 등의 신호)
- 헷갈리는 포인트 / 오해 방지
- 과제/읽을거리/다음 예고 (언급된 것만)

#### 2차 요약 — 통합

1차 요약 전체를 합쳐서 하나의 학습 문서로 재구성한다.

**프롬프트 출력 구조**:
- 전체 목차 재구성
- 중복 제거
- 흐름/의존관계 반영 (선행개념 → 후행개념)
- 정의/정리/예제를 재배치하여 "학습용 문서"로 완성

#### 액션 추출 — 과제/시험/일정

요약과 별도로, 전사 전문에서 **행동이 필요한 항목**을 추출한다.

**탐지 대상**:

| 유형 | 키워드/패턴 예시 |
|------|-----------------|
| 과제 | "과제", "레포트", "제출", "다음 주까지", "숙제" |
| 시험 | "시험", "퀴즈", "중간고사", "기말", "범위" |
| 일정 | "다음 주", "휴강", "보강", "변경", "취소" |
| 공지 | "공지", "이클래스", "확인해라", "올려놓을게" |

**출력 포맷** (노트에 삽입 + 데일리 노트 연동):
```markdown
## 액션 아이템
- [ ] **[과제]** 3장 연습문제 5, 7, 12번 — 다음 주 화요일까지 제출
- [ ] **[시험]** 중간고사 범위: 1~5장 — 4월 15일
- [ ] **[일정]** 다음 주 목요일 휴강
```

### 노트 생성 (obsidian_writer)

**출력 경로**: `{vault}/3_Areas/Lectures/{과목명}/YYYY-MM-DD_{과목명}.md`

**마크다운 템플릿**:

## Phase 2: Visual Anchors + RAG + 시험 대비

> Phase 2부터 **포트폴리오**를 겸한다. 코드 품질·문서화·테스트를 강화.

### Visual Anchors

전사 텍스트에서 **시각 자료에 의존하는 구간**을 자동 탐지하고 앵커 체크리스트를 생성한다.

**탐지 키워드**: "칠판", "판서", "보면", "그림", "표", "수식", "식", "여기", "이거", "이렇게", "저기", "슬라이드", "다이어그램", "그래프"

**앵커 후보 생성**:
- 키워드 탐지 시 해당 세그먼트 ±30초 범위를 앵커 후보로 지정
- 주변 전사 텍스트 스니펫(30~60초 분량) 자동 첨부
- 노트의 `## Visual Anchors` 섹션에 체크리스트로 출력

```markdown
## Visual Anchors
### 후보 (자동 생성)
- [ ] **[35:20]** "여기 칠판에 쓴 식을 보면…"
  > 주변 텍스트: "이항 계수를 전개하면... 여기 칠판에 쓴 식을 보면 n choose k가..."
  > 첨부: (없음)
- [ ] **[41:05]** "표로 정리하면…"
  > 주변 텍스트: "이걸 표로 정리하면 시간복잡도가 확 보이는데..."
  > 첨부: (없음)
```

**SRT/VTT 출력**: 전사 결과를 타임스탬프 포함 자막 파일로도 저장. 나중에 녹화 영상과 매칭 가능.

### 강의 컨텍스트 RAG

동일 과목의 이전 강의 요약을 벡터 DB에 저장하고, 새 강의 요약 시 컨텍스트로 활용한다.

**동기**: 교수님이 "지난 시간에 했던 거" 라고 말할 때, 이전 강의 내용을 참조하여 더 정확한 요약 가능.

**설계**:
- 강의 요약 완료 시 자동으로 임베딩 생성 → 로컬 벡터 DB 저장
- 새 강의 요약 프롬프트에 이전 강의 요약 중 관련 부분을 컨텍스트로 주입
- 벡터 DB: ChromaDB (로컬, 설치 간단, 파일 기반)
- 임베딩 모델: OpenAI text-embedding-3-small (또는 로컬 모델 검토)
- 과목별로 컬렉션 분리

**기대 효과**:
- "지난 시간 DFS" → 이전 강의에서 DFS 요약 부분을 자동 참조
- 학기 진행에 따라 누적 컨텍스트 축적 → 요약 품질 점진적 향상
- 강의 간 선수 개념 관계 자동 파악

### 시험 대비 A4

통합 요약을 기반으로 **시험 직전 5분 복습용 압축 문서**를 생성한다.

**프롬프트 출력 구조**:
- 암기할 것 (정의, 공식, 조건)
- 이해할 것 (원리, 흐름, 비교)
- 문제풀이 포인트 (접근법, 유형)
- 자주 나오는 함정 3개
- 5분 초압축 10줄

### eclass_crawler 연동

동일 워크스페이스의 `eclass_crawler` 프로젝트와 **느슨한 결합(Loose Coupling)** 으로 연동한다.
lesson-assist는 eclass_crawler의 출력 디렉토리(`output/`)만 읽으며, 직접 크롤링하지 않는다.

**연동 포인트**:

| 데이터 | 활용 | 구현 |
|--------|------|------|
| 실라버스 주차별 계획 | 날짜 → 주차 주제 자동 파악 → 요약 프롬프트에 주입 | `eclass.py` |
| 다운로드 강의자료 (PDF/PPT) | RAG 컨텍스트로 저장 → 요약 시 참조 | `rag/store.py` |
| 캘린더 이벤트 | 과제/시험 마감일 교차검증 | `eclass.py` |
| 게시판 공지 | 해당 과목 최신 공지 → 노트 첨부 | `eclass.py` |

**설정** (`config.yaml`):
```yaml
eclass:
  enabled: true
  data_dir: "G:\\CS_Study\\eclass_crawler\\output"
  course_mapping:
    자료구조: "자료구조(01)"
    알고리즘: "알고리즘(01)"
```

`course_mapping`은 lesson-assist에서 사용하는 과목명과 eclass 과목명이 다를 때 매핑한다.

### 온라인 수업 지원

오프라인/온라인 수업 모두 동일한 파이프라인으로 처리한다.

**입력 흐름**:
- 오프라인: 녹음 파일 (m4a/mp3/wav) → 기존 파이프라인
- 온라인: 화면 녹화 영상 (mp4/mkv/webm) → ffmpeg 오디오 추출 → 기존 파이프라인

**온라인 수업의 추가 이점**:
- 영상에서 Visual Anchor 시점의 **스크린샷 자동 추출** 가능 (판서 촬영 불필요)
- 오디오 품질이 더 좋음 (디지털 음성)

**CLI**:
```bash
# 오프라인 수업 (녹음)
python -m lesson_assist process --audio "lecture.m4a" --course "자료구조"

# 온라인 수업 (영상)
python -m lesson_assist process --input "lecture.mp4" --course "자료구조"
```

### 사용 전략 — 맥락 보전의 3계층

수업 녹음만으로는 시각 자료(판서, 슬라이드 보충)의 맥락이 빠진다.
이 문제를 **자동(RAG) + 반자동(Visual Anchors) + 수동(학생 필기)** 3계층으로 해결한다.

| 계층 | 담당 | 보전하는 맥락 |
|------|------|---------------|
| **1. RAG 자동** | eclass PPT 텍스트 + 이전 강의 요약 | 슬라이드 원본 내용, 이전 강의 선수 개념 |
| **2. Visual Anchors 반자동** | 앵커 탐지 → 학생에게 가이드 | 교수가 PPT 위에 추가한 판서/보충 설명 |
| **3. 학생 수동** | 선택적 사진 촬영 / 간단 메모 | 앵커가 짚어준 구간 중 중요한 것만 보완 |

**핵심 원리**: eclass PPT가 수업의 뼈대이므로 RAG에 넣으면 슬라이드 내용은 자동 참조된다. Visual Anchors는 "PPT에 없는 교수님 추가분"을 짚어주는 역할이다. 학생은 모든 판서를 찍을 필요 없이, 앵커가 표시한 구간 중 핵심만 선택적으로 보완하면 된다.

### 전체 사용 플로우

```
┌─ 수업 전 ──────────────────────────────────────────────┐
│  eclass_crawler 실행 → 강의자료(PPT/PDF) 다운로드      │
│  lesson-assist가 자료 텍스트를 RAG에 자동 저장          │
│  (이전 강의 요약 + PPT 원문이 모두 검색 대상이 됨)     │
└──────────────────────────────────────────────────────────┘
                         ▼
┌─ 수업 중 ──────────────────────────────────────────────┐
│  오프라인: 녹음 + 중요 판서만 선택적 촬영              │
│  온라인:   화면 녹화 (스크린샷 자동 추출됨)            │
└──────────────────────────────────────────────────────────┘
                         ▼
┌─ 수업 후 ──────────────────────────────────────────────┐
│  lesson-assist 실행                                    │
│  → 전사 → 분할 → 각 파트마다 RAG 검색·주입 → 요약     │
│  → Visual Anchors (판서/보충 구간 표시)                │
│  → 노트 생성 → 요약을 RAG에 저장 → 데일리 연동        │
│                                                        │
│  학생: 노트의 앵커 체크리스트 검토                     │
│  → 사진 있으면 첨부 / 없으면 기억·필기로 보완          │
│  → PPT에 있는 내용은 이미 RAG가 처리했으므로 스킵      │
└──────────────────────────────────────────────────────────┘
                         ▼
┌─ 시험 대비 ────────────────────────────────────────────┐
│  lesson-assist exam --course "자료구조"                 │
│  → 누적 노트 기반 A4 1장 압축 문서 생성                │
└──────────────────────────────────────────────────────────┘
```

### Watch 모드

지정 폴더를 감시하여 새 녹음 파일이 들어오면 자동으로 파이프라인을 실행한다.

```bash
python -m lesson_assist watch \
  --inbox "D:\Recordings\inbox" \
  --vault "G:\CS_Study\The Record"
```

- watchdog 라이브러리로 폴더 감시
- 새 오디오 파일 감지 → 파일명에서 과목명 파싱 (규칙: `YYYY-MM-DD_과목명.확장자`)
- 파이프라인 자동 실행
- 교정 단계는 건너뛰거나 나중에 일괄 처리 (교정 대기 큐)

### 이미지 반자동 연결

판서/슬라이드 사진을 Visual Anchors에 자동 연결한다.

**파일명 규칙**:
```
YYYY-MM-DD_과목명_MMmSSs.jpg
예: 2026-03-03_자료구조_35m20s.jpg
```

**동작**:
1. 이미지 inbox 폴더 감시
2. 파일명에서 날짜, 과목명, 시간 파싱
3. 해당 강의 노트의 Visual Anchors에서 가장 가까운 앵커 찾기
4. 앵커에 `![[이미지파일명]]` 자동 삽입

### Resume 기능

파이프라인 실행 중 실패 시 중간 산출물부터 재개한다.

- 각 단계 완료 시 상태 파일(`{id}_state.json`) 저장
- `--resume` 플래그로 마지막 성공 단계 이후부터 재실행
- 상태: `transcribed` → `reviewed` → `segmented` → `summarized` → `written`

### 데일리 노트 자동 연동

#### 1. 공부 기록 섹션

```markdown
## 공부 기록
- 📖 [[2026-03-03_자료구조|자료구조 5강]] — 이진 탐색 트리, AVL 트리
```

#### 2. 액션 아이템 → Todo/일정 섹션

강의에서 추출한 액션 아이템을 데일리 노트의 적절한 섹션에 삽입:

| 액션 유형 | 데일리 노트 섹션 | 예시 |
|-----------|-----------------|------|
| 과제 | `## Todo` | `- [ ] 자료구조 3장 연습문제 5, 7, 12번 (마감: 03/10)` |
| 시험 | `## 일정` 테이블 | `\| 04-15 \| 자료구조 중간고사 (범위: 1~5장) \|` |
| 일정 변경 | `## 일정` 테이블 | `\| 03-10 \| 자료구조 휴강 \|` |

**규칙**:
- 데일리 노트가 없으면 `Templates/daily.md` 기반으로 생성 (Templater 문법 치환)
- 이미 있으면 해당 섹션에 **추가** (기존 내용 유지)
- 중복 방지: 동일 과목+동일 날짜의 항목이 이미 있으면 스킵

### 알려진 이슈 / 향후 튜닝

- **Visual Anchors 키워드 튜닝 필요**: 첫 실전 테스트(수리논술)에서 98개 앵커 생성 — 대부분 "보면" 키워드의 false positive. "보면"은 한국어 구어체에서 말습관으로 쓰여 시각 자료와 무관한 경우가 많다. 실제 강의 데이터를 더 수집한 뒤 키워드 목록과 병합 전략을 조정해야 한다. (앵커 기능 자체는 정상 작동, 정밀도 튜닝 문제)
- **ChromaDB Python 3.14 비호환**: pydantic v1이 Python 3.14에서 동작하지 않아 ChromaDB 초기화 실패. JSON 기반 fallback 벡터 저장소로 대체 구현.
